# Fenjue V2 Acceptance Test Matrix

## 使用规则

- 每条用例保存输入夹具、数据库快照、代码/政策版本、实际输出和证据路径。
- `P0` 任一失败都阻止影子运行；`P1` 任一失败都阻止候选生产版晋级。
- 统计验收只使用样本外结果，且与相同成本和成交门的基准比较。

## 阶段验收

| 阶段 | 必须通过 |
|---|---|
| 结构契约 | 全部 DB、UNIT P0；`integrity_check=ok`；`foreign_key_check` 零行 |
| 第一阶段账本 | 全部 LEDGER P0，人工/券商对账一致率100% |
| 第二阶段事件 | 全部 TIME、EVENT P0，冻结解除审计覆盖率100% |
| 第三阶段执行 | 全部 EXEC、POLICY P0，核心仓违规0次 |
| 第四阶段影子 | 全部 GOV P0/P1，至少20个完整交易日且跨至少两类市场状态 |

概率发布另需满足 `05_backtest_governance_and_shadow_run.md` 中的100个独立样本、正负类各20个、60个独立交易日和校准质量门槛。
